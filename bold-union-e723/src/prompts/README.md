# LLM Prompts Configuration

This directory contains system prompts for the various LLM interactions in the backend application. Storing prompts as external Markdown files ensures they remain easily editable without altering the application logic.

## Prompt Files

### 1. `database_generation.md`
- **Purpose**: Generates a valid PostgreSQL DDL schema matching a natural language description.
- **Workflow**: Invoked when handling `POST /admin/create-schema` with a natural language `prompt`.

### 2. `query_generation.md`
- **Purpose**: Translates a natural language question into a SELECT SQL query and a short, user-friendly explanation based on the active database schema.
- **Workflow**: Invoked when handling `POST /query` with a natural language `prompt`.

### 3. `result_formatter.md`
- **Purpose**: Formats the returned database rows and query details into a user-friendly conversational response.
- **Workflow**: Invoked after a dynamically generated query is successfully executed in `POST /query` to structure the final text answer.

### 4. `sql_validation.md`
- **Purpose**: Validates or auto-corrects any invalid SQL generated by the LLM before execution (e.g., if it violates security constraints or schema rules).
- **Workflow**: Invoked during both `POST /admin/create-schema` and `POST /query` flows if the initial SQL validation checks fail.

### 5. `schema_context.md`
- **Purpose**: Provides guidelines and context rules for interpreting and querying the active database schema.
- **Workflow**: Prepended to the system prompt in the query translation flow within `POST /query`.
