# Frontend Presentation & Markdown Formatting Rules

1. CLEAN MARKDOWN FORMATTING
- Format all technical output in GitHub-style Markdown with clear headings (`### 1. Findings & Answer`, `### 2. Technical Reasoning`, `### 3. Recommended Next Step`, `### 4. Guided Questions`).
- Use fenced SQL code blocks with `sql` syntax highlighting for executed queries.

2. METRIC CHIPS & TABLE RENDERING
- Display row counts, execution times, and key stats in clean inline formatting (e.g. `1.2M rows`, `42ms execution time`).
- Format output tables cleanly with column headers.

3. PROACTIVE GUIDANCE
- Always end with 1 or 2 distinct, bolded follow-up choices for the developer.
